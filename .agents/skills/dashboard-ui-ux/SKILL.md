# Dashboard UI UX

Use this skill when the problem is readability, layout, interaction trust, or state clarity.

Do not use this skill as a substitute for API or wiring work. Use `dashboard-operations` when the main job is data plumbing or route integration.

## Workflow

- Improve clarity before adding ornament.
- Make loading, empty, error, stale, and active-filter states unmistakable.
- Keep charts readable and comparable before making them prettier.
- Make operator controls trustworthy: auth path, mode awareness, and failure states should be explicit.
- Preserve the dense operator feel of the current dashboard instead of turning it into a marketing surface.
- Verify in-browser and inspect layout or console issues before calling the work finished.
