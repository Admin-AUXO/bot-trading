# Dashboard UI UX

## Use When

- the issue is readability, layout, interaction trust, or state clarity

## Rules

- Improve clarity before ornament.
- Make loading, empty, error, stale, and active-filter states unmistakable.
- Keep charts comparable before making them prettier.
- Preserve the dense operator feel; this is a trading desk, not a marketing surface.
- Verify in-browser before calling it done.
