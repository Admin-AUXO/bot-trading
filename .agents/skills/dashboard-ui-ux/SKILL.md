# Dashboard UI UX

Use this skill for visual and interaction-focused dashboard work.

## Goals
- Improve clarity before adding ornament.
- Make states readable: loading, empty, error, active filters, stale data.
- Keep charts legible and comparable.
- Verify in-browser and inspect console or layout issues.
- Make operator controls trustworthy: if a control is shown, its auth path, mode awareness, and failure states should be explicit and credible.

## Preferred Tools
- `chrome_devtools` for layout, console, and rendering checks.
- `browsermcp` for real interaction flows.
- `filesystem` and `serena` for implementation context.
- `context7` for framework-specific UI behavior.
