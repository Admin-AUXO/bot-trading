# Dashboard UI UX

## Use When

- the issue is readability, layout, interaction trust, or state clarity

## Rules

- Improve clarity before ornament.
- Make loading, empty, error, stale, and active-filter states unmistakable.
- Keep charts comparable before making them prettier.
- Preserve the dense operator feel; this is a trading desk, not a marketing surface.
- In Discovery Studio, keep package filters editable through structured controls first; expose the full repo-supported filter surface, including relative-time fields and grouped sort options, before falling back to raw JSON.
- Keep page headers scan-friendly with stacked title and description, then actions and one dense stat strip.
- Verify dashboard UX work in browser on the real compose dashboard, and include dashboard build plus backend build when the surface depends on backend data.
- Verify in-browser before calling it done.
