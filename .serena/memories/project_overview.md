Purpose: Solana memecoin trading bot with backend trading engine plus Next.js dashboard. Stack: TypeScript ESM on Node.js, Express API, Prisma 7 with PostgreSQL, BullMQ/Redis, Next.js 16 dashboard, Solana integrations via Helius/Birdeye/Jupiter/Jito. Main working area is trading_bot/. Key safety rules: use logger utility instead of console, preserve circuit breaker/retry/rate-limit patterns around external APIs, keep control-plane writes behind bearer auth, prefer minimal reversible changes, avoid standalone Prisma migration files, and be especially careful in backend/src/core, backend/src/strategies, backend/src/services, and backend/prisma. Backend entrypoint is trading_bot/backend/src/index.ts; API server in trading_bot/backend/src/api/server.ts; dashboard in trading_bot/dashboard/. Historical analytics must come from immutable trade/snapshot history, and dashboard query keys must stay aligned with request params and backend filters.
