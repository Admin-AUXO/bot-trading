Purpose: Solana memecoin trading workspace with the application under `trading_bot/`: a TypeScript ESM/Node backend plus a Next.js 16 dashboard. Main stack: Express API, Prisma 7 with PostgreSQL 16, Solana integrations via Helius, Birdeye, Jupiter, Dex Screener, and Jito tip/config accounting. Runtime wiring is in-process and interval-driven. Redis is still provisioned in compose and env templates, but the checked-in app code does not currently instantiate a Redis client or BullMQ queue path. Key safety rules: use the shared logger instead of `console`, keep provider traffic inside `backend/src/services/`, preserve retry/rate-limit/cache/circuit-breaker patterns, keep control-plane writes behind bearer auth, avoid standalone Prisma migrations, and treat `backend/src/core`, `backend/src/strategies`, `backend/src/services`, and `backend/prisma` as safety-critical. Provider quota is a first-class runtime constraint: budget math is centralized in `backend/src/core/api-budget-manager.ts` plus `backend/src/config/provider-plan.ts`, non-essential work degrades first, and exits/reconciliation stay essential. `MarketRouter` now centralizes routed price, breadth, and seed reads so exit monitoring, outcome backfills, S1 wallet-activity pricing, S2 seed discovery, and S3 discovery do not bolt on quota-blind provider calls. Backend entrypoint is `trading_bot/backend/src/index.ts`; runtime assembly lives in `trading_bot/backend/src/bootstrap/runtime.ts`; API server lives in `trading_bot/backend/src/api/server.ts`; stats worker bootstrap lives in `trading_bot/backend/src/core/stats-aggregator.ts` and the SQL-heavy aggregator logic lives in `trading_bot/backend/src/workers/stats-aggregator.ts`; dashboard code lives in `trading_bot/dashboard/`. Historical analytics must come from immutable trade/position/snapshot rows. Dashboard chrome reflects the active runtime scope, while analysis pages can inspect other lanes through filters. `/quota` is the provider-runway page: service totals are global budgets, endpoint concentration can narrow by lane metadata, and mixed runtime/analysis scope must stay labeled honestly.
