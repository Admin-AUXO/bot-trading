Windows-oriented project commands. From `trading_bot/backend/`: `npm run dev`, `npm run build`, `npm run typecheck`, `npm run db:generate`, `npm run db:push`, `npm run db:views`, `npm run db:setup`, `npm run db:seed`, `npm run db:studio`, plus targeted tests like `node --env-file=.env.example --test --import tsx src/services/market-router.test.ts`. Use `npm run db:setup` when the database must match both Prisma tables and SQL views. From `trading_bot/dashboard/`: `npm run dev`, `npm run build`, `npm run lint`, `npm run start`. Serena in Codex is launched by the client with `uvx --from git+https://github.com/oraios/serena serena start-mcp-server --context codex --project-from-cwd`; use the attached Serena tools during tasks and only run maintenance from the repo root when repo shape or Serena-facing config changes: `uvx --from git+https://github.com/oraios/serena serena project index .` then `uvx --from git+https://github.com/oraios/serena serena project health-check .`. Useful Windows shell commands: `git status`, `git diff --stat`, `rg -n pattern path`, `Get-ChildItem`, `npm`, `node`.
