# Graphify

Purpose: keep the repo graph local, reproducible, and code-only.

## Supported Path

- Skill: [`.agents/skills/graphify/SKILL.md`](../../.agents/skills/graphify/SKILL.md)
- Wrapper: [`.codex/scripts/graphify.mjs`](../../.codex/scripts/graphify.mjs)
- Local runner: [`.codex/scripts/graphify-local-run.py`](../../.codex/scripts/graphify-local-run.py)
- Rebuild helper: [`.codex/scripts/graphify-rebuild.mjs`](../../.codex/scripts/graphify-rebuild.mjs)
- Ignore rules: [`.graphifyignore`](../../.graphifyignore)

## Commands

```bash
node ./.codex/scripts/graphify.mjs build-local .
node ./.codex/scripts/graphify.mjs query "RiskEngine"
node ./.codex/scripts/graphify.mjs hook status
node ./.codex/scripts/graphify-rebuild.mjs
```

## Rules

- The supported repo workflow is code-only. Do not promise semantic note ingestion, image ingestion, or delegated doc analysis.
- Read the most compact output (`GRAPH_QUICKREF.md`) for fast orientation. Use `GRAPH_MAPS.md` for cross-boundary queries, `GRAPH_SKILLS.md` for skill selection, `GRAPH_ACTIONS.md` for mutation catalogs, and `GRAPH_WORKFLOWS.md` for data flows.
- Verify any graph claim against source code before repeating it in docs or chat.
- Build with `build-local` when no graph exists yet. Rebuild only when `graphify-out/graph.json` already exists.
- The Node wrappers are the supported path on both macOS and Windows. The `.sh` files remain as POSIX compatibility shims.

## Outputs

| File | Purpose | LLM Priority |
|------|---------|--------------|
| `GRAPH_QUICKREF.md` | Ultra-compact node/edge summary | High |
| `GRAPH_MAPS.md` | Cross-boundary ownership maps | High |
| `GRAPH_SKILLS.md` | Skill selector guide | High |
| `GRAPH_ACTIONS.md` | State mutation catalog | High |
| `GRAPH_WORKFLOWS.md` | Data flow diagrams | High |
| `GRAPH_CLIENTS.md` | External client directory | Medium |
| `GRAPH_REPORT_COMPACT.md` | Token-optimized markdown (~50% size reduction) | High |
| `GRAPH_SCHEMA.json` | Machine-readable schema with compact nodes/edges/communities | High |
| `graph.json` | Raw NetworkX graph export for programmatic queries | Medium |
| `manifest.json` | File manifest | Low |

> Note: `graph-map-generator.mjs` regenerates `GRAPH_QUICKREF.md`. `GRAPH_REPORT.md`, `graph.html`, `cost.json`, and `cache/` are deleted after each run.

## Optimization Pipeline

The graph generation runs a multi-agent optimization pipeline:

1. **DataPruner** - removes duplicate Community Hub entries, normalizes verbose labels
2. **SchemaOptimizer** - creates structured JSON schema with semantic types
3. **SemanticEnricher** - adds cross-community bridges, betweenness centrality, semantic tags

**For LLM context:** Use `GRAPH_SCHEMA.json` for programmatic access or `GRAPH_QUICKREF.md` for quick scanning.

## Notes

- The wrapper provisions a local `.graphify-venv/` and records the interpreter in `.graphify_python` plus `graphify-out/.graphify_python`.
- Graph artifacts and local graphify state are intentionally gitignored.
- Compact outputs are generated by `.codex/scripts/graph_optimize.py`.
